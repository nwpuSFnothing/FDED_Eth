# 工程使用与测试指令参考

本文档记录当前 `FDED_Eth` / `FDED_host` 工程的常用运行、测试和热指纹表验证指令。

## 1. 基本目录

推荐在 Host 工程目录下执行 Python 命令：

```powershell
cd C:\Users\14854\Desktop\FDED_Eth\FDED_host
```

当前默认网络参数：

```text
FPGA IP: 192.168.0.2
Host IP: 192.168.0.3
UDP Port: 8080
```

如实际网络配置不同，在命令中加入：

```powershell
--fpga-ip <FPGA_IP> --host-ip <HOST_IP> --port <PORT>
```

## 2. 单包 SHA-256 链路测试

从 `FDED_host` 目录测试 FPGA SHA-256 UDP 链路：

```powershell
python -B ..\FDED_test\test_udp_sha256_2kb.py --case abc
```

运行完整测试集：

```powershell
python -B ..\FDED_test\test_udp_sha256_2kb.py --suite
```

预期：

- `result pass`
- digest 与 Host `hashlib.sha256()` 一致
- 未写入热表时，普通数据一般返回 `status 0x00`

## 3. FPGA 热表基础测试

`SHA256("abc")` 的 digest 为：

```text
ba7816bf8f01cfea414140de5dae2223b00361a396177a9cb410ff61f20015ad
```

写入 slot0 并测试命中：

```powershell
python -B main.py clear-hot-table
python -B main.py write-hot-digest --slot 0 --digest-hex ba7816bf8f01cfea414140de5dae2223b00361a396177a9cb410ff61f20015ad
python -B ..\FDED_test\test_udp_sha256_2kb.py --case abc
```

预期：

```text
status 0x01
```

清空后再次测试：

```powershell
python -B main.py clear-hot-table
python -B ..\FDED_test\test_udp_sha256_2kb.py --case abc
```

预期：

```text
status 0x00
```

## 4. 单文件处理

基本格式：

```powershell
python -B main.py process-file <输入文件> [参数]
```

CDC 分块：

```powershell
python -B main.py process-file input.bin --chunk-mode cdc --min-size 512 --avg-size 1024 --max-size 1468 --verify-local
```

固定大小分块：

```powershell
python -B main.py process-file input.bin --chunk-mode fixed --fixed-size 1024 --verify-local
```

常用参数：

```text
--db-path <路径>        sqlite 指纹数据库路径
--verify-local         用 Host hashlib 复核 FPGA digest
--print-chunks         打印每个 chunk 的状态
--result-dir <目录>    输出 Markdown 和 PNG 报表
--write-dir <目录>     唯一 chunk 写出目录
```

## 5. 目录处理

基本格式：

```powershell
python -B main.py process-dir <输入目录> [参数]
```

示例：

```powershell
python -B main.py process-dir testdata --recursive --chunk-mode fixed --fixed-size 1024 --verify-local
```

注意：不要把输入目录设置为默认数据库所在的 `data`，否则可能把 `data/fingerprints.db` 当作测试文件处理。建议使用独立数据库路径：

```powershell
python -B main.py process-dir testdata --recursive --db-path Result\hot_test.db --verify-local
```

## 6. 加载 FPGA 热指纹表

从 sqlite 指纹表按 `ref_count` 选 top-N digest 写入 FPGA：

```powershell
python -B main.py load-hot-table --db-path Result\hot_test.db --limit 512
```

只清空热表：

```powershell
python -B main.py clear-hot-table
```

写入单条 digest：

```powershell
python -B main.py write-hot-digest --slot 0 --digest-hex <64位hex digest>
```

## 7. 阶段 4：Host 联动与统计

处理前自动加载热表：

```powershell
python -B main.py process-file input.bin --db-path Result\hot_test.db --load-hot-table --hot-limit 512 --verify-hot-hit --verify-local
```

处理中周期刷新热表：

```powershell
python -B main.py process-file input.bin --db-path Result\hot_test.db --load-hot-table --hot-limit 512 --hot-refresh-interval-s 10 --verify-hot-hit --verify-local
```

参数说明：

```text
--load-hot-table             处理开始前加载 FPGA 热表
--hot-limit 512              从 sqlite 选择 top-512 digest
--hot-refresh-interval-s 10  处理中每隔约 10 秒刷新一次 FPGA 热表
--verify-hot-hit             FPGA 返回 HOT_HIT 时，再查 sqlite 复核
--no-clear-hot-table         加载前不清空 FPGA 热表
```

统计字段说明：

```text
fpga_hot_hits   FPGA 返回 HOT_HIT 的 chunk 数
host_lookups    Host 实际执行 sqlite 判重查询的次数
lookup_saved    因 FPGA HOT_HIT 跳过的 Host 判重查询次数
hot_hit_ratio   fpga_hot_hits / chunks
hot_loaded      最近一次加载到 FPGA 的 digest 数
hot_refreshes   热表加载/刷新次数
```

## 8. 推荐热表有效性测试

生成重复数据：

```powershell
Remove-Item -Recurse -Force testdata -ErrorAction SilentlyContinue
New-Item -ItemType Directory testdata | Out-Null
python -c "from pathlib import Path; b=bytes([i%256 for i in range(1024)]); Path('testdata/a.bin').write_bytes(b*200); Path('testdata/b.bin').write_bytes(b*200)"
```

删除旧测试库：

```powershell
Remove-Item -Force Result\hot_test.db,Result\hot_test.db-wal,Result\hot_test.db-shm -ErrorAction SilentlyContinue
```

第一遍建库：

```powershell
python -B main.py process-dir testdata --recursive --db-path Result\hot_test.db --chunk-mode fixed --fixed-size 1024 --load-hot-table --hot-limit 512 --hot-refresh-interval-s 10 --verify-hot-hit --verify-local
```

第二遍验证热表命中：

```powershell
python -B main.py process-dir testdata --recursive --db-path Result\hot_test.db --chunk-mode fixed --fixed-size 1024 --load-hot-table --hot-limit 512 --hot-refresh-interval-s 10 --verify-hot-hit --verify-local --print-chunks
```

第二遍预期：

```text
fpga_hot_hits > 0
lookup_saved > 0
hot_hit_ratio 明显大于 0
```

如果第二遍仍然 `fpga_hot_hits 0`，优先回到第 3 节用 `abc` digest 测试 FPGA 热表写入和查询链路。

## 9. 当前 FPGA 热表更新时间

FPGA 热表只在 Host 发送 control packet 时更新：

```text
clear-hot-table
write-hot-digest
load-hot-table
process-file/process-dir --load-hot-table
--hot-refresh-interval-s 周期触发
```

普通 hash 请求只查询热表，不会自动插入新 digest。运行中 sqlite 的 `ref_count` 变化后，只有下一次 Host 主动加载或周期刷新时才会同步到 FPGA。
