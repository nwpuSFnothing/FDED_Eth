## 3. 当前 FPGA 热指纹表更新时间点

当前 FPGA 热指纹表不会在普通 hash 请求中自动插入新 digest。

它只在 Host 发送 control packet 时更新：

- `clear-hot-table`：清空 FPGA 热表 valid 位。
- `write-hot-digest`：写入指定 slot。
- `load-hot-table`：从 sqlite 读取 top-N digest 后逐项写入 FPGA。
- `process-file/process-dir --load-hot-table`：处理开始前加载一次。
- `--hot-refresh-interval-s N`：处理中每隔约 `N` 秒由 Host 主动刷新一次。

也就是说，运行中 sqlite 的 `ref_count` 改变后，FPGA 热表只有在下一次 Host 主动加载或周期刷新时才会同步。

## 4. 测试现象

执行：

```powershell
python -B main.py process-file sample.bin --load-hot-table --hot-limit 512 --hot-refresh-interval-s 10 --verify-hot-hit --verify-local --print-chunks
```

第一次报错原因是当前目录没有 `sample.bin`，不是协议或热表逻辑错误。

随后运行中曾看到类似统计：

```text
fpga_hot_hits 0
host_lookups  26
lookup_saved  0
hot_hit_ratio 0.0000
hot_loaded    512
hot_refreshes 1
```

含义：

- FPGA 热表加载动作执行过。
- 本次处理的 chunk digest 没有命中已加载的热表。
- 运行时间短于 10 秒，因此只发生开始时的 1 次加载，没有触发周期刷新。

后续执行：

```powershell
python -B main.py process-dir data --recursive --load-hot-table --hot-limit 512 --hot-refresh-interval-s 10 --verify-hot-hit --verify-local
```

实际处理的是：

```text
FDED_host/data/fingerprints.db
```

原因是输入目录 `data` 与默认 sqlite 数据库目录相同，程序把数据库文件本身当作测试数据处理。后续测试应将输入数据目录和数据库路径分开，例如把数据库放到 `Result/hot_test.db`。

## 5. 推荐测试方式

建议使用独立测试目录和独立数据库：

```powershell
cd C:\Users\14854\Desktop\FDED_Eth\FDED_host

Remove-Item -Recurse -Force testdata -ErrorAction SilentlyContinue
New-Item -ItemType Directory testdata | Out-Null
python -c "from pathlib import Path; b=bytes([i%256 for i in range(1024)]); Path('testdata/a.bin').write_bytes(b*200); Path('testdata/b.bin').write_bytes(b*200)"

Remove-Item -Force Result\hot_test.db,Result\hot_test.db-wal,Result\hot_test.db-shm -ErrorAction SilentlyContinue

python -B main.py process-dir testdata --recursive --db-path Result\hot_test.db --chunk-mode fixed --fixed-size 1024 --load-hot-table --hot-limit 512 --hot-refresh-interval-s 10 --verify-hot-hit --verify-local

python -B main.py process-dir testdata --recursive --db-path Result\hot_test.db --chunk-mode fixed --fixed-size 1024 --load-hot-table --hot-limit 512 --hot-refresh-interval-s 10 --verify-hot-hit --verify-local --print-chunks
```

第二遍重点观察：

```text
fpga_hot_hits
lookup_saved
hot_hit_ratio
hot_refreshes
```

如果第二遍 `fpga_hot_hits > 0` 且 `lookup_saved > 0`，说明 FPGA 热表在去重流程中起作用。
