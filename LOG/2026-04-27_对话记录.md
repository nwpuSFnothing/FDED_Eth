## 2. 系统目标收敛

用户明确提出：

- 暂时不把分块放到 FPGA
- FPGA 第一阶段只实现 `SHA-256 hash`
- 目标是先实现：
  - 主机传输数据
  - FPGA 计算 hash
  - FPGA 返回 digest

据此将第一阶段系统收敛为：

- `UDP payload -> FPGA -> SHA-256 -> 32B digest -> UDP reply`

并约束为：

- `payload <= 55B`
- FPGA 自己做单块 SHA-256 padding
- 返回固定 `32B digest`

## 5. default_nettype 问题排查

在综合中出现了大量类似错误：

- `net type must be explicitly specified ... when default_nettype is none`

分析后确认：

- 问题不是旧以太网工程本身有错
- 而是新接入的 SHA-256 代码把 ``default_nettype none`` 带进了工程
- 导致旧工程中的老式端口声明方式被连带报错

最终确定策略：

- **不改旧工程风格**
- **让新代码适配旧工程**

执行的处理包括：

- 恢复旧文件 `smi_config.v` 的老式写法
- 去掉工程内 `sha256_core.v / sha256_w_mem.v / sha256_k_constants.v / sha256_stream.v` 中对旧代码有外溢影响的 `default_nettype none`
